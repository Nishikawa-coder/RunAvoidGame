## 見どころ
### 最後までプレイヤーの興味をひくような物語の構成
本作のステージは全部で四つあり、各ステージをクリアする毎に「丸」に変化が起きる。しかし、「丸」が手に入れるパーツは人間の物だけでなく動物の部位や物だったりもする。本当に人間になっていくのか最後にならないと分からない構成にする事で、ユーザーに好奇心をそそるような内容にしている。
### 音楽と背景による明確なオチ
物語の最後に「丸」は頭を手に入れる。頭を手に入れた途端に背景やBGM、敵の姿が一斉に変化する。これにより物語のクライマックスである事を示唆し、プレイヤーを惹き込む工夫をしている。
## 開発における工夫
### 多種多様な敵とステージ
敵やステージのバリエーションを増やす事でプレイヤーを飽きさせないようにした。上から落ちる敵や回転する敵、飛び跳ねる敵等ステージを進めていくにつれ敵の動きを増やしていった。またステージについても絶えず変化をつけていった。この時、他のゲーム作品も参考にするようにした。
### こだわりのUI・UX
主人公の動き、カメラ、表示等にこだわった。主人公の操作については最初「丸」であった事から滑るような動きにしようとした。しかし、それでは難易度が高くプレイヤーにとってストレスになると考えて滑らないようにした。カメラについては主人公に追随するようにしていたが、上下に動くと映像が見づらいと感じ上下の動きは固定するようにした。表示については初めて自分のゲームをプレーする人が操作などに戸惑う部分がないようにその都度指示を表示するようにした。
## 課題
### バグ修正
ライフが一気に2つ減る、主人公が壁にくっつく、ゲームをビルドした時のキャラクターの動きが違うなど。
### 難易度の調整
主人公の移動速度やジャンプの高さ、動く床の動きがわかりづらいなど。
